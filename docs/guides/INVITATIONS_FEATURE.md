# 📨 ميزة إرسال الدعوات - دليل الاستخدام

## ✅ تم التطبيق بنجاح!

تم إضافة نظام متكامل لإرسال دعوات تفعيل الحسابات للأعضاء في لوحة الإدارة.

---

## 🎯 الميزات المضافة

### 1. زر إرسال دعوة لكل عضو 📧

في بطاقة كل عضو، ستجد:
- **زر أخضر** (✉️) لإرسال دعوة تفعيل - يظهر فقط للأعضاء غير المفعلين
- **زر أخضر** (✓) مع علامة صح - يظهر للأعضاء المفعلين (غير قابل للنقر)

### 2. زر إرسال دعوات جماعية 📢

في أعلى قسم "أعضاء النادي"، ستجد زر **"إرسال دعوات جماعية"** باللون الأخضر.

---

## 📖 طريقة الاستخدام

### الطريقة 1: إرسال دعوة لعضو واحد

1. **انتقل إلى** قسم "أعضاء النادي"
2. **ابحث عن العضو** المطلوب
3. **اضغط على زر المغلف الأخضر** (✉️) في أعلى يسار البطاقة
4. **ستظهر نافذة** تحتوي على:
   - معلومات العضو (الاسم، البريد، اللجنة)
   - رابط التفعيل
   - تاريخ انتهاء الصلاحية (7 أيام)
   - تعليمات الإرسال
5. **اضغط "نسخ الرابط"** لنسخ الرابط تلقائياً
6. **أرسل الرابط للعضو** عبر واتساب أو بريد إلكتروني

#### مثال على الرابط:
```
https://www.adeeb.club/members/activate.html?token=a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

---

### الطريقة 2: إرسال دعوات جماعية

#### أ) لجميع الأعضاء غير المفعلين

1. **اضغط على** زر "إرسال دعوات جماعية" (الأخضر)
2. **اختر** "جميع الأعضاء غير المفعلين"
3. **اضغط "متابعة"**
4. **أكد الإرسال** - سيظهر لك عدد الأعضاء المستهدفين
5. **انتظر** حتى يتم إنشاء الدعوات
6. **اضغط "عرض استعلام SQL"** للحصول على روابط الدعوات

#### ب) حسب اللجنة

1. **اضغط على** زر "إرسال دعوات جماعية"
2. **اختر** "حسب اللجنة"
3. **اختر اللجنة** من القائمة المنسدلة
4. **اضغط "متابعة"**
5. **أكد الإرسال**
6. **اضغط "عرض استعلام SQL"** للحصول على الروابط

---

### الطريقة 3: الحصول على روابط الدعوات من قاعدة البيانات

بعد إنشاء الدعوات الجماعية:

1. **انسخ استعلام SQL** من النافذة المنبثقة
2. **افتح Supabase SQL Editor**
3. **الصق الاستعلام** وشغّله
4. **ستحصل على جدول** يحتوي على:
   - الاسم
   - البريد
   - اللجنة
   - رابط التفعيل
   - تاريخ الانتهاء

#### مثال على الاستعلام:

```sql
-- الحصول على روابط الدعوات المنشأة حديثاً
SELECT 
  m.full_name as "الاسم",
  m.email as "البريد",
  m.committee as "اللجنة",
  'https://www.adeeb.club/members/activate.html?token=' || mi.invitation_token as "رابط_التفعيل",
  mi.expires_at as "تاريخ_الانتهاء"
FROM member_invitations mi
JOIN members m ON m.id = mi.member_id
WHERE mi.status = 'pending' 
AND mi.created_at > NOW() - INTERVAL '1 hour'
ORDER BY m.committee, m.full_name;
```

5. **صدّر النتائج** إلى Excel أو CSV
6. **أرسل لكل عضو رابطه الخاص**

## 🔒 الأمان والقيود

### القيود المطبقة

1. ✅ **لا يمكن إرسال دعوة** لعضو فعّل حسابه مسبقاً
2. ✅ **البريد الإلكتروني مطلوب** لإنشاء الدعوة
3. ✅ **الدعوة صالحة لـ 7 أيام فقط**
4. ✅ **كل دعوة تُستخدم مرة واحدة فقط**
5. ✅ **التوكن عشوائي وآمن** (UUID v4)

### الصلاحيات

- ✅ فقط **الإداريون** يمكنهم إرسال الدعوات
- ✅ يتم تسجيل **معرف الإداري** الذي أنشأ الدعوة
- ✅ **RLS محكم** على جدول `member_invitations`

## 📊 تتبع الدعوات

### عرض جميع الدعوات المعلقة

```sql
SELECT 
  m.full_name as الاسم,
  mi.email as البريد,
  mi.status as الحالة,
  mi.created_at as تاريخ_الإنشاء,
  mi.expires_at as تاريخ_الانتهاء,
  CASE 
    WHEN mi.expires_at < NOW() THEN '❌ منتهية'
    ELSE '✅ صالحة'
  END as الصلاحية
FROM member_invitations mi
JOIN members m ON m.id = mi.member_id
WHERE mi.status = 'pending'
ORDER BY mi.created_at DESC;
```

### عرض الدعوات المستخدمة

```sql
SELECT 
  m.full_name as الاسم,
  mi.email as البريد,
  mi.accepted_at as تاريخ_التفعيل
FROM member_invitations mi
JOIN members m ON m.id = mi.member_id
WHERE mi.status = 'accepted'
ORDER BY mi.accepted_at DESC;
```

### حذف الدعوات المنتهية

```sql
-- تحديث حالة الدعوات المنتهية
UPDATE member_invitations
SET status = 'expired'
WHERE status = 'pending'
AND expires_at < NOW();

-- أو حذفها نهائياً
DELETE FROM member_invitations
WHERE status = 'expired'
AND expires_at < NOW() - INTERVAL '30 days';
```

## 🐛 استكشاف الأخطاء

### المشكلة: الزر لا يظهر

**الحل:**
1. تأكد من تحديث `admin.html`
2. امسح الكاش (Ctrl + Shift + R)
3. تأكد من تحميل `admin.js` الجديد

### المشكلة: خطأ عند إنشاء الدعوة

**الأسباب المحتملة:**
- جدول `member_invitations` غير موجود
- صلاحيات RLS تمنع الإدراج
- البريد الإلكتروني فارغ

**الحل:**
```sql
-- التحقق من وجود الجدول
SELECT * FROM member_invitations LIMIT 1;

-- التحقق من الصلاحيات
SELECT * FROM pg_policies WHERE tablename = 'member_invitations';
```

### المشكلة: الرابط لا يعمل

**الحل:**
1. تأكد من أن الرابط كامل ويبدأ بـ `https://`
2. تأكد من عدم انتهاء صلاحية الدعوة
3. تأكد من عدم استخدام الرابط مسبقاً
