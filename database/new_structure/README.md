# الهيكلة الجديدة للمجلس الإداري والأعضاء

## 📋 نظرة عامة

هذه الهيكلة الجديدة تفصل بوضوح بين:
1. **أعضاء النادي** (`members`) - جميع أعضاء النادي
2. **المستخدمين الإداريين** (`admin_users`) - من لديهم صلاحيات الوصول للوحة التحكم
3. **المجالس الإدارية** (`board_councils`) - الحاضنة التي تضم الأعضاء الإداريين
4. **مناصب المجلس** (`board_positions`) - ربط الأعضاء بمناصبهم في المجالس

---

## 🏗️ الهيكل الهرمي

```
members (أعضاء النادي)
    ↓
    ├─→ admin_users (مستخدمون إداريون - صلاحيات النظام)
    │
    └─→ board_councils (المجالس الإدارية - حاضنة)
            ↓
        board_positions (مناصب المجلس - الرئيس، النائب، إلخ)
```

---

## 📊 الجداول بالتفصيل

### 1. `members` - أعضاء النادي
**الغرض**: يحتوي على جميع أعضاء النادي (موجود مسبقاً)

**الحقول الرئيسية**:
- `id` - معرف العضو
- `full_name` - الاسم الكامل
- `email` - البريد الإلكتروني
- `avatar_url` - صورة العضو
- `user_id` - ربط مع Supabase Auth

---

### 2. `admin_users` - المستخدمون الإداريون
**الغرض**: إدارة صلاحيات الوصول للوحة التحكم فقط

**الحقول الرئيسية**:
- `id` - معرف المستخدم الإداري
- `member_id` - ربط مع جدول members (كل admin_user هو member)
- `user_id` - ربط مع Supabase Auth
- `role` - الدور: `super_admin`, `admin`, `moderator`
- `permissions` - صلاحيات مفصلة بصيغة JSON
- `is_active` - حالة الحساب الإداري

**الأدوار**:
- **Super Admin**: كل الصلاحيات (إدارة المستخدمين الإداريين، الإعدادات، إلخ)
- **Admin**: معظم الصلاحيات (لا يمكنه إدارة المستخدمين الإداريين)
- **Moderator**: صلاحيات محدودة (إدارة المحتوى فقط)

**مثال على الصلاحيات**:
```json
{
  "dashboard": true,
  "members": {"read": true, "create": true, "update": true, "delete": false},
  "board": {"read": true, "create": true, "update": true, "delete": false},
  "news": {"read": true, "create": true, "update": true, "delete": false}
}
```

**الدوال المساعدة**:
- `check_admin_permission(user_id, resource, action)` - للتحقق من صلاحية معينة

---

### 3. `board_councils` - المجالس الإدارية
**الغرض**: الحاضنة التي تضم الأعضاء الإداريين (مثل: "المجلس الإداري 2024")

**الحقول الرئيسية**:
- `id` - معرف المجلس
- `title` / `title_ar` - عنوان المجلس
- `description` / `description_ar` - وصف المجلس
- `start_date` / `end_date` - الفترة الزمنية
- `is_active` - هل المجلس نشط
- `is_current` - هل هذا المجلس الحالي (واحد فقط)
- `is_visible` - هل يظهر في الموقع العام
- `display_order` - ترتيب العرض

**الميزات**:
- ✅ مجلس حالي واحد فقط (يتم تطبيقه تلقائياً)
- ✅ إمكانية عرض مجالس سابقة
- ✅ تتبع الفترة الزمنية لكل مجلس

**الدوال المساعدة**:
- `get_current_council()` - للحصول على المجلس الحالي
- `transition_to_new_council(new_council_id, end_current)` - للانتقال لمجلس جديد

**View**:
- `board_councils_with_stats` - عرض المجالس مع عدد الأعضاء

---

### 4. `board_positions` - مناصب المجلس
**الغرض**: ربط الأعضاء بمناصبهم في المجالس الإدارية

**الحقول الرئيسية**:
- `id` - معرف المنصب
- `council_id` - ربط مع المجلس
- `member_id` - ربط مع العضو
- `position_type` - نوع المنصب (من قائمة محددة)
- `position_title` / `position_title_ar` - عنوان المنصب
- `position_rank` - ترتيب المنصب (1 = الأعلى)
- `department` / `department_ar` - القسم/اللجنة (اختياري)
- `bio` / `bio_ar` - نبذة عن العضو في هذا المنصب
- `custom_avatar_url` - صورة مخصصة للمنصب (اختياري)
- `social_links` - روابط التواصل الاجتماعي
- `is_visible` - هل يظهر في الموقع العام
- `display_order` - ترتيب العرض

**أنواع المناصب** (`position_type`):
1. `president` - الرئيس (ترتيب 1)
2. `vice_president` - نائب الرئيس (ترتيب 2)
3. `ceo` - الرئيس التنفيذي (ترتيب 3)
4. `leader` - القائد (ترتيب 4)
5. `vice_leader` - نائب القائد (ترتيب 5)
6. `secretary` - الأمين العام (ترتيب 6)
7. `treasurer` - أمين الصندوق (ترتيب 7)
8. `member` - عضو مجلس (ترتيب 99)

**الميزات**:
- ✅ تعيين تلقائي للعناوين والترتيب حسب نوع المنصب
- ✅ إمكانية تخصيص العناوين
- ✅ دعم الأقسام/اللجنات
- ✅ صورة مخصصة لكل منصب (اختياري)
- ✅ روابط تواصل اجتماعي مخصصة

**القيود**:
- عضو واحد لكل مجلس (لا يمكن للعضو أن يكون له أكثر من منصب في نفس المجلس)

**الدوال المساعدة**:
- `get_current_board_members()` - للحصول على أعضاء المجلس الحالي
- `get_member_current_position(member_id)` - للحصول على منصب عضو معين

**View**:
- `board_positions_detailed` - عرض المناصب مع معلومات الأعضاء والمجالس

## 🔄 السيناريوهات الشائعة

### 1. إضافة عضو جديد للنادي
```sql
-- 1. إضافة العضو في جدول members
INSERT INTO members (full_name, email, ...)
VALUES ('أحمد محمد', 'ahmed@example.com', ...);
```

### 2. منح عضو صلاحيات إدارية
```sql
-- 2. إضافة العضو كمستخدم إداري
INSERT INTO admin_users (member_id, user_id, role)
VALUES (
    'member_uuid',
    'auth_user_uuid',
    'admin'
);
-- الصلاحيات ستُعيّن تلقائياً حسب الدور
```

### 3. إنشاء مجلس إداري جديد
```sql
-- 3. إنشاء المجلس
INSERT INTO board_councils (
    title, 
    title_ar, 
    start_date, 
    is_current
)
VALUES (
    'Board Council 2024',
    'المجلس الإداري 2024',
    '2024-01-01',
    true
);
-- سيتم إلغاء تعيين المجالس الأخرى كـ current تلقائياً
```

### 4. تعيين عضو في منصب بالمجلس
```sql
-- 4. تعيين العضو في منصب
INSERT INTO board_positions (
    council_id,
    member_id,
    position_type
)
VALUES (
    'council_uuid',
    'member_uuid',
    'president'
);
-- العنوان والترتيب سيُعيّنان تلقائياً
```

### 5. الحصول على أعضاء المجلس الحالي
```sql
-- 5. عرض أعضاء المجلس الحالي
SELECT * FROM get_current_board_members();
```

## 🔐 الأمان (RLS)

### `admin_users`
- ✅ المستخدمون الإداريون يقرأون بياناتهم
- ✅ Super Admins يقرأون ويديرون كل المستخدمين الإداريين

### `board_councils`
- ✅ الجميع يقرأ المجالس المرئية
- ✅ المصادقون يقرأون كل المجالس
- ✅ الإداريون يديرون المجالس

### `board_positions`
- ✅ الجميع يقرأ المناصب المرئية في المجالس المرئية
- ✅ المصادقون يقرأون كل المناصب
- ✅ الإداريون يديرون المناصب
