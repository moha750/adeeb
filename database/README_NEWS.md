## جدول قاعدة البيانات

### جدول `news`

```sql
CREATE TABLE news (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  summary TEXT,
  image_url TEXT,
  author_id UUID REFERENCES auth.users(id),
  author_name TEXT,
  status TEXT DEFAULT 'draft' CHECK (status IN ('draft', 'published', 'archived')),
  views INTEGER DEFAULT 0,
  is_featured BOOLEAN DEFAULT false,
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### الحقول

| الحقل | النوع | الوصف |
|------|------|-------|
| `id` | UUID | المعرف الفريد للخبر |
| `title` | TEXT | عنوان الخبر (مطلوب) |
| `content` | TEXT | محتوى الخبر الكامل (يدعم HTML) |
| `summary` | TEXT | ملخص قصير للخبر (يتم إنشاؤه تلقائياً من أول 75 حرف) |
| `image_url` | TEXT | رابط صورة الخبر |
| `author_id` | UUID | معرف الكاتب |
| `author_name` | TEXT | اسم الكاتب الأول (للتوافق مع الإصدارات القديمة) |
| `authors` | TEXT[] | مصفوفة أسماء الكُتّاب (يدعم كُتّاب متعددين) |
| `status` | TEXT | حالة الخبر (draft/published/archived) |
| `views` | INTEGER | عدد المشاهدات |
| `is_featured` | BOOLEAN | هل الخبر مميز |
| `published_at` | TIMESTAMPTZ | تاريخ النشر |
| `created_at` | TIMESTAMPTZ | تاريخ الإنشاء |
| `updated_at` | TIMESTAMPTZ | تاريخ آخر تحديث |

## حالات الخبر

1. **draft (مسودة)**: خبر قيد الإعداد، غير مرئي للزوار
2. **published (منشور)**: خبر منشور ومرئي للجميع
3. **archived (مؤرشف)**: خبر قديم تم أرشفته

## الأمان

### Row Level Security (RLS)

تم تفعيل RLS على جدول `news` مع السياسات التالية:

1. **Allow public read published news**: السماح للجميع بقراءة الأخبار المنشورة
2. **Allow authenticated read all news**: السماح للمصادقين بقراءة جميع الأخبار
3. **Allow authenticated insert/update/delete**: السماح للمصادقين بإضافة/تعديل/حذف الأخبار

### تنظيف النصوص

يتم استخدام دالة `safe()` في admin.js لتنظيف النصوص من HTML tags الخطيرة عند العرض.

## إصلاح المشاكل

### مشكلة 1: `sb is not defined` في لوحة التحكم

إذا ظهرت رسالة خطأ في كونسول لوحة التحكم:
```
Error loading news: ReferenceError: sb is not defined
```

**السبب**: دوال إدارة الأخبار كانت خارج نطاق IIFE الرئيسي حيث يتم تعريف متغير `sb` (Supabase client).

**الحل**: تم نقل جميع دوال إدارة الأخبار داخل IIFE الرئيسي في ملف `admin.js` (السطر 10530+).

**التحديثات المطبقة**:
1. نقل قسم NEWS MANAGEMENT داخل IIFE
2. إصلاح المسافات البادئة لجميع الدوال
3. جعل دوال `editNews` و `deleteNews` متاحة عالمياً عبر `window`
4. إضافة دالة `showNotification` لعرض الإشعارات
5. إضافة أنيميشن CSS للإشعارات

### مشكلة 2: `sb is not defined` في صفحة الأخبار

إذا ظهرت رسالة خطأ في كونسول صفحة الأخبار:
```
Error loading news: ReferenceError: sb is not defined
```

**السبب**: ملف `news.js` لم يقم بتعريف متغير `sb` من `window.sbClient`.

**الحل**: إضافة السطر التالي في بداية `news.js`:
```javascript
const sb = window.sbClient;
```

### مشكلة 3: `Cannot read properties of null` في script.js

إذا ظهرت رسالة خطأ:
```
Uncaught TypeError: Cannot read properties of null (reading 'addEventListener')
```

**السبب**: ملف `script.js` يحاول الوصول إلى عناصر HTML غير موجودة في صفحة الأخبار (مثل `#menuToggle`).

**الحل**: إضافة فحص للتأكد من وجود العناصر قبل استخدامها:
```javascript
if (menuToggle && nav) {
  menuToggle.addEventListener("click", function () {
    // ...
  });
}
```

**الملفات المُحدثة**:
- `news.js`: إضافة `const sb = window.sbClient;`
- `script.js`: إضافة فحوصات `if (menuToggle && nav)` في 4 أماكن

## إعداد Supabase Storage

لتفعيل رفع الصور، يجب تشغيل الكود التالي في Supabase SQL Editor:

```sql
-- إنشاء bucket للصور
INSERT INTO storage.buckets (id, name, public)
VALUES ('news-images', 'news-images', true)
ON CONFLICT (id) DO NOTHING;

-- السماح للجميع بقراءة الصور
CREATE POLICY "Allow public read access"
ON storage.objects FOR SELECT
USING (bucket_id = 'news-images');

-- السماح للمستخدمين المسجلين برفع الصور
CREATE POLICY "Allow authenticated users to upload images"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'news-images' 
  AND auth.role() = 'authenticated'
);
```

أو تشغيل ملف `news_storage.sql` مباشرة.
