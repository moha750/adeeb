# دليل ربط فورم التواصل والنشرة البريدية بلوحة التحكم

## نظرة عامة
تم ربط نموذجي "تواصل معنا" و"انضم إلى مجتمعنا" من الموقع الرئيسي بلوحة التحكم الإدارية مع حفظ البيانات في **Supabase فقط**.

> ✅ **تم إلغاء الحفظ في Google Sheets والاعتماد على Supabase بشكل كامل**

---

## 📋 الخطوات المطلوبة

### 1. إنشاء الجداول في Supabase

قم بتنفيذ ملف SQL التالي في Supabase SQL Editor:

```bash
database/contact_and_newsletter_tables.sql
```

**الجداول المُنشأة:**
- `contact_messages` - لحفظ رسائل التواصل
- `newsletter_subscriptions` - لحفظ اشتراكات النشرة البريدية

**الميزات:**
- ✅ Row Level Security (RLS) مفعّل
- ✅ سياسات أمان: السماح بالإدراج للجميع، القراءة والتحديث للمصادقين فقط
- ✅ فهارس لتحسين الأداء
- ✅ تحديث تلقائي لـ `updated_at`
- ✅ قيود على الحالات (status constraints)

---

### 2. التحديثات على الموقع الرئيسي

#### ملف: `script.js`

**نموذج التواصل:**
- تم تحديث معالج الإرسال ليكون `async`
- يحفظ البيانات في جدول `contact_messages`
- معالجة الأخطاء مع رسائل واضحة

**نموذج النشرة البريدية:**
- تم تحديث معالج الإرسال ليكون `async`
- يحفظ البيانات في جدول `newsletter_subscriptions`
- معالجة حالة البريد المكرر (error code 23505)

---

### 3. التحديثات على لوحة التحكم

#### ملف: `admin/admin.html`

**القائمة الجانبية:**
- ✅ إضافة رابط "رسائل التواصل" 📧
- ✅ إضافة رابط "النشرة البريدية" 📰

**الأقسام الجديدة:**

**أ) قسم رسائل التواصل (`#section-contact-messages`):**
- إحصائيات: إجمالي الرسائل، جديدة، مقروءة، تم الرد، مؤرشفة
- فلترة حسب الحالة
- جدول عرض الرسائل مع التفاصيل
- إمكانية عرض وتحديث كل رسالة

**ب) قسم النشرة البريدية (`#section-newsletter`):**
- إحصائيات: إجمالي الاشتراكات، نشط، ملغي
- فلترة حسب الحالة
- جدول عرض الاشتراكات
- تصدير البريد الإلكتروني للمشتركين النشطين

#### ملف: `admin/admin.js`

**الدوال المضافة:**

**رسائل التواصل:**
- `loadContactMessages()` - تحميل الرسائل من Supabase
- `renderContactMessagesStats()` - عرض إحصائيات الرسائل
- `renderContactMessages(filter)` - عرض جدول الرسائل مع الفلترة
- `viewContactMessage(id)` - عرض وتحديث رسالة محددة

**النشرة البريدية:**
- `loadNewsletterSubscriptions()` - تحميل الاشتراكات من Supabase
- `renderNewsletterStats()` - عرض إحصائيات الاشتراكات
- `renderNewsletterList(filter)` - عرض جدول الاشتراكات مع الفلترة
- تصدير البريد الإلكتروني كملف نصي

**Event Listeners:**
- زر التحديث لكل قسم
- فلترة حسب الحالة
- تصدير البريد الإلكتروني

## 🎨 الميزات

### رسائل التواصل
- **الحالات المتاحة:**
  - 🆕 جديدة (new) - اللون الأزرق
  - 👁️ مقروءة (read) - اللون الأخضر
  - 💬 تم الرد (replied) - اللون البنفسجي
  - 📦 مؤرشفة (archived) - اللون الرمادي

- **البيانات المحفوظة:**
  - الاسم الكامل
  - البريد الإلكتروني (قابل للنقر لفتح البريد)
  - الموضوع (اختياري)
  - نص الرسالة
  - الحالة
  - ملاحظة إدارية (اختياري)
  - تاريخ الإنشاء والتحديث

### النشرة البريدية
- **الحالات المتاحة:**
  - ✅ نشط (active) - اللون الأخضر
  - ❌ ملغي (unsubscribed) - اللون الأحمر

- **البيانات المحفوظة:**
  - البريد الإلكتروني (فريد)
  - الحالة
  - تاريخ الاشتراك
  - تاريخ الإلغاء (إن وجد)

- **ميزة التصدير:**
  - تصدير جميع البريد الإلكتروني النشط
  - تنسيق نصي (كل بريد في سطر)
  - اسم الملف: `newsletter-emails-YYYY-MM-DD.txt`

## 🔒 الأمان

### Row Level Security (RLS)
- ✅ **الإدراج (INSERT):** متاح للجميع (anon, authenticated) - للسماح بإرسال النماذج من الموقع
- ✅ **القراءة (SELECT):** متاح للمصادقين فقط - الإداريين في لوحة التحكم
- ✅ **التحديث (UPDATE):** متاح للمصادقين فقط - لتحديث حالة الرسائل

### معالجة الأخطاء
- ✅ معالجة البريد المكرر في النشرة البريدية
- ✅ رسائل خطأ واضحة للمستخدم
- ✅ تسجيل الأخطاء في Console

## 🎯 تجربة المستخدم

### في الموقع الرئيسي:
1. المستخدم يملأ نموذج "تواصل معنا" أو "انضم إلى مجتمعنا"
2. عند الإرسال، يتم حفظ البيانات في Supabase
3. رسالة نجاح/فشل واضحة باستخدام SweetAlert2
4. إعادة تعيين النموذج بعد النجاح

### في لوحة التحكم:
1. الإداري يفتح قسم "رسائل التواصل" أو "النشرة البريدية"
2. يشاهد الإحصائيات والبيانات في جداول منظمة
3. يمكنه فلترة البيانات حسب الحالة
4. يمكنه عرض وتحديث كل رسالة
5. يمكنه تصدير البريد الإلكتروني للمشتركين

## 🚀 الاستخدام

### للمطورين:
1. تأكد من تشغيل أوامر SQL في Supabase
2. تأكد من أن Supabase Client مهيأ بشكل صحيح
3. اختبر النماذج في الموقع الرئيسي
4. تحقق من ظهور البيانات في لوحة التحكم

### للإداريين:
1. سجل الدخول إلى لوحة التحكم
2. انتقل إلى "رسائل التواصل" أو "النشرة البريدية"
3. استخدم الفلاتر لتصفية البيانات
4. اضغط على "عرض" لرؤية تفاصيل الرسالة
5. حدّث الحالة وأضف ملاحظات إدارية
6. صدّر البريد الإلكتروني عند الحاجة

## 🔧 استكشاف الأخطاء

### المشكلة: لا تظهر البيانات في لوحة التحكم
**الحل:**
- تأكد من تشغيل أوامر SQL في Supabase
- تحقق من سياسات RLS
- افتح Console وتحقق من الأخطاء

### المشكلة: فشل إرسال النموذج
**الحل:**
- تحقق من اتصال Supabase
- تأكد من صحة سياسات RLS للإدراج
- تحقق من صحة البيانات المدخلة

### المشكلة: البريد المكرر
**الحل:**
- هذا سلوك متوقع - البريد الإلكتروني فريد في جدول النشرة
- سيظهر للمستخدم رسالة "مسجل مسبقاً"
