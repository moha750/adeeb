# تحديث إحصائيات الأعضاء - إضافة الحسابات المفعلة وغير المفعلة

## نظرة عامة

تم إضافة إحصائيات جديدة لقسم أعضاء النادي لعرض عدد الحسابات المفعلة والحسابات غير المفعلة.

---

## التحديثات المطبقة

### 1️⃣ إضافة الإحصائيات في JavaScript

**الملف:** `admin.js`

**التغييرات:**
```javascript
// Count activated vs non-activated accounts
const activated = members.filter(m => m.user_id && m.account_status === 'active').length;
const notActivated = total - activated;
const activatedPercent = total > 0 ? Math.round((activated * 100) / total) : 0;
const notActivatedPercent = total > 0 ? Math.round((notActivated * 100) / total) : 0;
```

**معايير الحساب المفعل:**
- يجب أن يحتوي على `user_id` (مرتبط بحساب Supabase Auth)
- يجب أن تكون `account_status = 'active'`

**معايير الحساب غير المفعل:**
- أي عضو لا يستوفي شروط الحساب المفعل

### 2️⃣ إضافة كروت العرض

#### كارت الحسابات المفعلة:

**الألوان:**
- خلفية: تدرج أزرق فاتح (#dbeafe → #bfdbfe)
- حدود: أزرق (#93c5fd)
- نص: أزرق داكن (#1e40af)
- أيقونة: `fa-user-check` ✓

#### كارت الحسابات غير المفعلة:

```html
    <div class="stat-number" style="color:#b45309">
      [عدد] ([نسبة]%)
    </div>
```

**الألوان:**
- خلفية: تدرج أصفر (#fef3c7 → #fde68a)
- حدود: أصفر (#fcd34d)
- نص: برتقالي داكن (#b45309)
- أيقونة: `fa-user-clock` ⏰

### 3️⃣ تحديث التنسيق CSS

**الملف:** `members-stats.css`

#### ترتيب الكروت الجديد:

1. **الصف الأول:** إجمالي الأعضاء (عرض كامل)
2. **الصف الثاني:** بيانات مكتملة + بيانات غير مكتملة (نصف ونصف)
3. **الصف الثالث:** حسابات مفعلة + حسابات غير مفعلة (نصف ونصف) ← **جديد**
4. **الصفوف التالية:** إحصائيات اللجان (4 كروت في كل صف)

#### تحديثات الشاشات الكبيرة (1100px+):
```css
/* حسابات مفعلة وغير مفعلة في صف واحد */
.members-stats-grid > .card:nth-child(4) {
  grid-column: 1 / 3; /* يأخذ نصف العرض */
}

.members-stats-grid > .card:nth-child(5) {
  grid-column: 3 / 5; /* يأخذ النصف الثاني */
}

/* البقية (اللجان) توزع بشكل عادي */
.members-stats-grid > .card:nth-child(n+6) {
  grid-column: span 1;
}
```

#### تحديثات الشاشات المتوسطة (768px - 1100px):
```css
/* حسابات مفعلة وغير مفعلة في صف واحد */
.members-stats-grid > .card:nth-child(4),
.members-stats-grid > .card:nth-child(5) {
  grid-column: span 1;
}

/* البقية (اللجان) توزع بشكل عادي */
.members-stats-grid > .card:nth-child(n+6) {
  grid-column: span 1;
}
```

#### الشاشات الصغيرة (أقل من 768px):
- جميع الكروت تأخذ عرض كامل (عمود واحد)

## الفوائد

### 1. **رؤية واضحة للحسابات**
- معرفة فورية لعدد الحسابات المفعلة
- تتبع الأعضاء الذين لم يفعلوا حساباتهم بعد

### 2. **تحسين المتابعة**
- سهولة تحديد الأعضاء الذين يحتاجون لإرسال دعوات
- متابعة معدل تفعيل الحسابات

### 3. **إحصائيات دقيقة**
- نسب مئوية واضحة
- أرقام محلية بالعربية

### 4. **تصميم متناسق**
- ألوان متناسقة مع باقي الكروت
- تنسيق responsive لجميع الشاشات

## أمثلة على الحالات

### مثال 1: نادي جديد
```
إجمالي الأعضاء: 50
حسابات مفعلة: 5 (10%)
حسابات غير مفعلة: 45 (90%)
```
**التفسير:** النادي في بداية تفعيل الحسابات

### مثال 2: نادي نشط
```
إجمالي الأعضاء: 200
حسابات مفعلة: 180 (90%)
حسابات غير مفعلة: 20 (10%)
```
**التفسير:** معظم الأعضاء فعّلوا حساباتهم

### مثال 3: نادي متوسط
```
إجمالي الأعضاء: 100
حسابات مفعلة: 60 (60%)
حسابات غير مفعلة: 40 (40%)
```
**التفسير:** نصف الأعضاء تقريباً فعّلوا حساباتهم

1. افتح لوحة التحكم

3. تحقق من ظهور الكروت الجديدة:

5. جرب على شاشات مختلفة (كبيرة، متوسطة، صغيرة)

## ملاحظات تقنية

### معايير الحساب المفعل:
```javascript
m.user_id && m.account_status === 'active'
```

- `user_id`: يجب أن يكون موجود (مرتبط بـ Supabase Auth)
- `account_status`: يجب أن يكون 'active'

### الحالات الممكنة لـ account_status:
- `pending`: قيد الانتظار
- `active`: نشط (مفعل)
- `suspended`: معلق
- `inactive`: غير نشط

## التطويرات المستقبلية المقترحة

1. **فلتر حسب حالة التفعيل**
   - إضافة خيار في الفلاتر لعرض المفعلين فقط أو غير المفعلين

2. **إجراءات سريعة**
   - زر "إرسال دعوات للحسابات غير المفعلة"
   - زر "تذكير الأعضاء بتفعيل الحساب"

3. **تقارير تفصيلية**
   - تقرير بأسماء الأعضاء غير المفعلين
   - تاريخ آخر دعوة مرسلة

4. **إشعارات تلقائية**
   - تنبيه عند انخفاض معدل التفعيل
   - تذكير دوري للأعضاء غير المفعلين
